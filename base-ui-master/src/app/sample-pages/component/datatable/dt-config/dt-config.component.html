Columns Definition Configuration:

Datatable header can be configured as per below using columns attribute. This attribute is an array of columns properties. The properties are as below:-

  • field - string - Data field that we want to display in column. If data object didn't contain given field name, empty column will be rendered
  • headerText - string - Column header title
  • width - string/number - column width. Can be either pixel (number) or percentage (string)
  • allowSorting - boolean - enable column sorting by clicking the header.By default, all columns are sortable
  • visible - boolean - Show/hide particular column. By default all columns are visible
  • headerTextAlign - string - Header text alignment. Can be either 'Left'/'Right'/'Center'/'Justify'. By default it is left align
  • textAlign - string - Column data text alignment. Can be either 'Left'/'Right'/'Center'/'Justify'. By default it is left align
  • columnTemplate - object - rendering column to specific template. Accept an object with id(template id) and context(template model)
  • render - function - A method for specific column rendering. Contain a callback function with follo parameters callback(data:particula cell data, cell: particular cell element, row: particular row data)
  • type - string - Define the type for the column. Can be defined to any. However, defined it to below keywords will render specific column template
    • checkbox
    • radio
    • expandable
            dtConfig = {{ '{' }}
              columns: [
                {{ '{' }} field: 'name', headerText: 'Name', width: '30%' {{ '}' }}
                {{ '{' }} field: 'SlNo', headerText: 'No.', width: '10%' {{ '}' }}
                {{ '{' }} field: 'gender' {{ '}' }},
                {{ '{' }} field: 'age', headerText: 'Age' {{ '}' }}
              ]
            {{ '}' }}

Pagination Setting Configuration

Datatable pagination can also be configured by setting tableInit initialPage property. initialPage has 3 properties as below:

  1. pageSize : The number of rows that will be display per page. Default value is 15
  2. pageSizes : This is to determine the pagination per page number in the dropdown. it can take a boolean or an array of pagination number. Default value is [10, 15, 25, 50, 100]. To turn off this feature, simply set the value to false
  3. pageCount : This will determine the number of page that pagination number will display before the ellipsis. Default value is 4

In the example, we turn off paging dropdown function by set pageSizes to false. We also set pageSize to 7 and pageCount to 8

            {{ '{' }}
              columns: [{{ '{' }} field: 'SlNo' }{{ '}' }} {{ '{' }} field: 'name' }{{ '}' }} {{ '{' }} field: 'gender' }{{ '}' }} {{ '{' }} field: 'company' {{ '}' }}],
              initialPage: {{ '{' }}
                pageSize: 7,
                pageSizes: false,
                pageCount: 8
              {{ '}' }}
            {{ '}' }};

In example below, we set pageSizes to [3, 6, 9, 12, 15] while pageSize and pageCount to default values

          {{ '{' }}
            columns: [{{ '{' }} field: 'SlNo' }{{ '}' }} {{ '{' }} field: 'name' }{{ '}' }} {{ '{' }} field: 'gender' }{{ '}' }} {{ '{' }} field: 'company' {{ '}' }}],
            initialPage: {{ '{' }}
              pageSize: 7,
              pageSizes: [3, 6, 9, 12, 15]
              pageCount: 8
            {{ '}' }}
          {{ '}' }};

Label Configuration

Datatable allows user to dynamically set the display label for the table by using tableInit property of displayText

DisplayText has properties as below:

  1. emptyRecord : The text should be displayed in the table if there is no data. Default is 'No data available in table'
  2. pagerDropDown : Label for pager dropdown. The text format should be 'A #pager B'. The word #pager represents the dropdown and it is mandatory. if the word #pager is missing, the dropdown it will not display the pager dropdwon. Default is 'Display #pager record per page'
  3. currentPageInfo : Label for showing current page. Default 'Showing page {{ '{' }}0{{ '}' }} of {{ '{' }}1{{ '}' }}'. {{ '{' }}0{{ '}' }} will be interpolated by datatable as current page number and {{ '{' }}1{{ '}' }} will be the total page
  4. totalItemsInfo : Total items label. Default is none

In the example, we configure the datatable as below:

            {{ '{' }}
              columns: [{{ '{' }} field: 'name', headerText: 'Name', width: '30%'  {{ '}' }}],
              displayText: {{ '{' }}
                emptyRecord: 'This is empty record display',
                pagerDropDown: '#pager records in one page',
                currentPageInfo: 'Display {{ '{' }}0 {{ '}' }} of {{ '{' }}1 {{ '}' }}',
                totalItemsInfo: '(Total {{ '{' }}0 {{ '}' }} rows in all pages)'
              {{ '}' }}
            {{ '}' }};

Text Wrap

By default, text in datatable are not wrapped. They are truncated with ellipsis. To wrap the text in datatable, we can use textWrap property in Datatable configuration. textWrap is an object with 2 attributes:-

  • allowWrap - boolean - Allow datatable text to wrap. By default, it is false
  • mode - string - Wrapping mode for datatable. Can be either these 3 values:-
    • Both - Wrap both header and content texts
    • Header - Wrap header text only
    • Content - Wrap content text only

In example below, we wrap both header and content using mode Content

        {{ '{' }}
          columns: [
            {{ '{' }} field: 'SlNo', headerText: 'No.', width: '20%' {{ '}' }},
            {{ '{' }} field: 'name', headerText: 'Name', width: '30%' {{ '}' }},
            {{ '{' }} field: 'age', headerText: 'Age', width: '20%' {{ '}' }},
            {{ '{' }} field: 'gender', headerText: 'Gender', width: '20%' {{ '}' }},
            {{ '{' }} field: 'company', headerText: 'Company', width: '20%' {{ '}' }}
          ],
          textWrap: {{ '{' }}
            allowWrap: true,
            mode: 'Both'
          {{ '}' }}
        {{ '}' }};
